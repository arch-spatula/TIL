## 간만에 퇴근 후 개발

- 이번에는 유튜브를 잠시 차단하고 개발을 했습니다.
- 묘하게 행복했습니다. 회사에서는 개발과 사무업무 중간 성격의 작업을 계속하고 정보처리기사 공부를 하면서 개발을 할 시간이 별로 없었습니다.
- 중요한 것이 성취감인지 잘 모르겠습니다.
- 본격적으로 작업하기 전에는 오히려 조금 불안했습니다. 작업을 시작하고서는 오히려 괜찮아졌습니다.
- 내일도 이어서 이런 작업을 비슷하게 하면 될 것 같습니다.
- 원하는 것이 너무 높은 것 같습니다. 실제 중요한 것은 문제를 해결하는 것인데 자꾸 멀리 설계하려고 합니다.
- 개인적으로 어려운 문제를 해결해야 하는데 지금 하는 작업이 간만에 난이도가 있었을 정도로 개발 같은 개발을 안 하고 있언 것 같습니다.
- 침묵 혹은 지루함 그 자체는 꽤 중요한 것 같습니다.
- 필요없는 정신적인 자극에 기대어서 정작 중요한 생각들을 안 했던 것 같습니다.
- 유튜브 차단을 오랫동안 유지하는 것이 필요할 것 같습니다.

## 라이브 DB를 드랍한 주니어 개발자의 최후

- https://www.youtube.com/watch?v=lpZzBUuLjgI
- 운영 DB를 Drop 하면 시말서입니다. 사람을 기르고 싶다면 위로가 아니라 시말서와 경위서를 올리도록 해야 합니다.
  - 운영 DB 권한은 정상적이면 보통 줍니다. 이것은 당연히 책임감 있는 개발자들이 대부분일 경우입니다.
  - 확인을 대충하는 사람들 때문에 운영 DB 권한을 다 제거합니다.
- 일 못하는 사람들일수록 절차가 많이 필요합니다. 일 잘하는 사람일수록 절차는 줄여야 합니다.
- 운영 DB 권한을 주는 회사가 절대 이상한 것이 아니라 운영 DB를 Drop하는 개발자가 잘 못하는 것입니다.
  - 불신도 실력이라고 본다면 불신하지 않은 회사의 잘못이라고 생각할 수 있습니다.
  - 더 큰 문제는 남탓하는 것과 배우지 않는 자세가 문제입니다.
  - 이런 사고를 친 주니어 개발자는 더 좋은 곳으로 이직 후에 짧게 다니다가 짤렸다고 합니다.
- 인간의 기억력과 감은 상당히 부정확합니다.
- 중요한 것을 확인하고 집중하고 안 중요한 것은 덜 신경쓰는 방식으로 처리해야 합니다. 이것은 최소 중 최소 일머리입니다.
